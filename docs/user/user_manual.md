# 基金套利监控应用 - 用户手册

## 概述

基金套利监控应用是一款基于Python PyQt6开发的桌面应用程序，专门用于监控ETF（交易所交易基金）和LOF（上市开放式基金）的套利机会。应用提供实时数据监控、套利机会识别、数据可视化分析等功能。

### 主要功能

- **实时监控**：监控ETF和LOF基金的净值与市场价格差异
- **套利计算**：自动计算套利收益率，考虑交易费用
- **数据可视化**：提供价格历史、收益率趋势、价差分布等图表
- **筛选过滤**：支持按基金类型、价差范围、机会等级等条件筛选
- **数据管理**：本地SQLite数据库存储历史数据
- **数据导出**：支持将数据导出为CSV格式

### 适用场景

- 基金投资者寻找套利机会
- 金融研究人员分析基金定价效率
- 量化交易员监控市场异常
- 金融教育工作者演示套利原理

## 安装说明

### 系统要求

- **操作系统**：Windows 10/11, macOS 10.15+, Linux (支持GUI)
- **Python版本**：Python 3.9 或更高版本
- **内存**：至少4GB RAM
- **磁盘空间**：至少500MB可用空间

### 方法一：从源代码运行（开发环境）

1. **克隆或下载项目代码**
   ```bash
   git clone <仓库地址>
   cd fund-arbitrage-monitor
   ```

2. **创建虚拟环境（推荐）**
   ```bash
   python -m venv venv

   venv\Scripts\activate

   source venv/bin/activate
   ```

3. **安装依赖包**
   ```bash
   pip install -r requirements.txt
   ```

4. **运行应用**
   ```bash
   python -m src.main
   ```

## 功能详解

### 数据刷新

#### 手动刷新
1. 点击工具栏的"刷新数据"按钮或按F5键
2. 状态栏显示"正在刷新数据..."
3. 刷新完成后，界面自动更新
4. 状态栏显示刷新结果和发现的套利机会数量

#### 刷新过程
1. 从数据源获取最新基金数据
2. 计算套利机会
3. 保存数据到本地数据库
4. 更新所有界面组件

### 套利计算原理

应用采用专业的套利计算模型：

#### ETF套利计算
```
溢价套利（价格 > 净值）：
  1. 申购ETF份额（现金 → ETF份额）
  2. 在二级市场卖出ETF份额（ETF份额 → 现金）

折价套利（价格 < 净值）：
  1. 在二级市场买入ETF份额（现金 → ETF份额）
  2. 赎回ETF份额（ETF份额 → 现金）

费用考虑：
  - 佣金：0.03%
  - 印花税：0.1%
  - 总费用：0.13%
```

#### LOF套利计算
```
溢价套利（价格 > 净值）：
  1. 场外申购LOF份额
  2. 转托管到场内
  3. 在场内卖出

折价套利（价格 < 净值）：
  1. 在场内买入LOF份额
  2. 转托管到场外
  3. 场外赎回

费用考虑：
  - 申购费：1.5%
  - 赎回费：0.5%
  - 总费用：2.0%
```

#### 机会等级判定
- **强机会**：价差 > 费用成本 × 1.5
- **弱机会**：费用成本 < 价差 ≤ 费用成本 × 1.5
- **无机会**：价差 ≤ 费用成本

### 图表分析

#### 价格历史图表
- **用途**：分析基金净值与市场价格的关系
- **解读**：
  - 当价格线在净值线上方时，存在溢价
  - 当价格线在净值线下方时，存在折价
  - 两线距离越大，套利空间越大

#### 收益率趋势图表
- **用途**：跟踪套利收益率随时间的变化
- **解读**：
  - 正值表示正向套利机会
  - 负值表示反向套利机会
  - 趋势向上表示机会增强，趋势向下表示机会减弱

#### 价差分布图表
- **用途**：了解所有基金价差的分布情况
- **解读**：
  - 显示价差的集中区域
  - 识别异常值（极高或极低价差）
  - 分析市场整体套利机会状况

### 数据筛选

#### 按基金类型筛选
- **ETF**：交易所交易基金，T+0交易，费用较低
- **LOF**：上市开放式基金，可跨市场套利，费用较高

#### 按价差范围筛选
- **负价差**：折价状态，价格低于净值
- **正价差**：溢价状态，价格高于净值
- **零价差**：价格与净值基本一致

#### 按机会等级筛选
- **强机会**：优先关注的高收益机会
- **弱机会**：需要谨慎评估的边际机会
- **无机会**：当前没有套利空间的基金

### 数据管理

#### 本地数据库
应用使用SQLite数据库存储数据：
- **数据位置**：`data/fund_arbitrage.db`
- **存储内容**：
  - 基金基本信息
  - 历史价格数据
  - 套利机会记录
  - 系统配置

#### 数据导出
可将当前显示的数据导出为CSV格式：

2. 选择保存位置和文件名
3. 数据包含所有可见列

## 故障排除

### 常见问题

#### 1. 应用无法启动
- **可能原因**：缺少依赖包或Python版本不兼容
- **解决方案**：
  - 确认Python版本 ≥ 3.9：`python --version`
  - 重新安装依赖：`pip install -r requirements.txt`
  - 检查虚拟环境是否激活

#### 2. 界面显示异常
- **可能原因**：PyQt6兼容性问题或DPI缩放设置
- **解决方案**：
  - 检查系统DPI设置
  - 尝试在高DPI兼容模式下运行
  - 更新PyQt6到最新版本

#### 3. 数据刷新失败
- **可能原因**：网络连接问题或数据源异常
- **解决方案**：
  - 检查网络连接
  - 切换数据源（如从真实API切换到模拟数据）
  - 查看日志文件获取详细错误信息

#### 4. 图表不显示数据
- **可能原因**：数据格式问题或图表组件初始化失败
- **解决方案**：
  - 确认已刷新数据
  - 检查数据是否包含必要字段
  - 尝试切换图表类型

#### 5. 数据库错误
- **可能原因**：数据库文件损坏或权限问题
- **解决方案**：
  - 检查数据库文件权限
  - 删除损坏的数据库文件（应用会自动重建）
  - 位置：`data/fund_arbitrage.db`

### 日志文件
应用生成日志文件，位置：`logs/app.log`
- **查看日志**：`tail -f logs/app.log`
- **日志级别**：INFO（正常）、WARNING（警告）、ERROR（错误）

### 术语解释
- **ETF**：交易所交易基金，可在证券交易所买卖
- **LOF**：上市开放式基金，可在交易所和场外市场交易
- **净值 (NAV)**：基金资产净值除以基金份额总数
- **市场价格**：基金在二级市场的交易价格
- **价差**：市场价格与净值的差异
- **套利**：利用同一资产在不同市场的价格差异获利
- **溢价**：价格高于净值的状态
- **折价**：价格低于净值的状态
